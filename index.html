Подготовка к новой схеме

Перенос рабочих аккаунтов в @mexcinsertbot

i
Сейчас продолжаем работать по старой схеме. Эта страница — только подготовка к переходу на новую схему через бота. В текущей работе ничего не меняется, пока не будет отдельной команды. Задача сейчас: занести все рабочие аккаунты в бота.

Что нужно сделать

  1. Отправить клиенту готовый текст — он включает API и присылает ключи (Шаг 1).
  2. Добавить аккаунт в @mexcinsertbot по схеме (Шаг 2).
  3. Бот выдаёт адрес, клиент вносит его в вайтлист — аккаунт добавлен.

Два варианта подключения

Предпочтительный — API + веб

Рекомендуется

API + веб

Клиент даёт веб-токен и API-ключ (Access + Secret) с полным доступом. Работает автовывод профита.

Запасной

Только веб

Клиент даёт только веб-токен. Автовывод не работает — профит выводится вручную.

1

Отправить клиенту текст

С этого начинается перенос

Скопируй сообщение и отправь клиенту. В нём — ссылка на видео и что прислать в ответ.

Привет. Переносим аккаунт, нужно два действия.

1. Настройка и API-ключ.
Видео: https://artemdost.github.io/mexc-setup/
До 0:41 — включи настройки безопасности и выпусти API-ключ с полным доступом (чтение, торговля, вывод).
Пришли мне: веб-токен (WEBID), Access Key, Secret Key. Secret показывается один раз при создании — скопируй сразу.

2. Адрес для вывода.
После 0:41 на видео — как добавить адрес в вайтлист вывода.
Сам адрес пришлю после того, как заведу аккаунт. Добавь его в вайтлист, сеть BSC (BEP-20).

После этого аккаунт готов.
→
В ответ клиент присылает веб-токен (WEBID), Access Key и Secret Key. Secret виден один раз при создании — клиент копирует его сразу.

2

Добавить аккаунт в бота

Когда клиент прислал данные

← схему можно листать вбок →

От клиента приходят три поля — загрузи их в @mexcinsertbot:

Веб-токен
строка вида WEB… (WEBID) — по ней бот находит UID аккаунта
Access Key
API-ключ MEXC (из видео, Шаг 1)
Secret Key
секрет API-ключа MEXC

Бот создаёт кошелёк и выдаёт адрес для вывода. Передай адрес клиенту — он вносит его в вайтлист вывода на MEXC (сеть BSC / BEP-20; это вторая часть видео, после 0:41). Затем в боте нажми «Проверить»: если адрес в вайтлисте — аккаунт добавлен.

3

Если клиент не даёт API

Запасной вариант — только веб-токен

Если клиент не выдаёт API, в боте при добавлении выбрать «Без API» и занести только веб-токен — аккаунт попадёт в базу.

!
Без API автовывод не работает: профит придётся выводить вручную. Предпочтительный вариант — полный доступ по API.

Что дальше

Пока работаем по старой схеме. Когда все аккаунты будут в @mexcinsertbot, перейдём на новую схему: профиль, список аккаунтов, обновление токенов и автовывод — через бота. О переходе будет отдельная команда.
